# util_rgbe_test.cpp - RGBE 颜色编码往返测试

## 概述

此文件测试 Cycles 工具库中 RGBE（RGB + Exponent，又称 Radiance HDR 格式）颜色编码的编解码正确性。RGBE 是一种紧凑的 HDR（高动态范围）颜色存储格式，用 4 字节存储一个 HDR 像素（RGB 各 8 位尾数 + 1 字节共享指数），广泛用于 HDR 环境贴图。

## 测试用例

### TEST(RGBE, round_trip)
- **功能**: 验证 `rgb_to_rgbe` 和 `rgbe_to_rgb` 之间的往返编解码精度。由于 RGBE 格式的有限精度，编码后再解码的结果与原始值会有量化误差。
- **验证场景**:
  - **常规 HDR 值**: `{7.334898, 5.811583, 2.414717}` -> 编码解码后 `{7.34375, 5.8125, 2.40625}`（量化到 RGBE 精度）
  - **低动态范围值**: `{0.0875, 0.0515, 0.2499}` -> 编码解码后保持近似值
  - **极端动态范围**: `{4e-6, 30257.0, 1.0}` -> 小值被量化为 0，大值近似保留
  - **零值**: `{0, 0, 0}` -> 编码解码后仍为零
  - **极小正值**: `{5.9e-8, 0, 0}` -> 低于 RGBE 可表示范围，编码为零
  - **RGBE 可表示最小值边界**: `{6.0e-8, 0, 0}` -> 编码为最小可表示值 `{1.192e-7, 0, 0}`
  - **负值处理**: `{-0.864, 0.559, -0.223}` -> RGBE 支持负值的近似表示
  - **浮点极值**: `{-FLT_MAX, FLT_MAX, 0}` -> 被钳制到 RGBE 可表示最大范围 `{-65280, 65280, 0}`
  - **无穷大处理**: `{inf, 127, 129}` -> 无穷大被钳制，其他值按比例调整
  - **NaN**: 未测试，属于未定义行为
