# magic.h - 魔法纹理 SVM 节点

## 概述
`magic.h` 实现了 Cycles 的魔法纹理节点(`NODE_TEX_MAGIC`)，生成具有迷幻视觉效果的程序化颜色图案。该纹理通过多层嵌套的三角函数（正弦和余弦）运算产生复杂的干涉条纹和色彩变化，深度参数控制嵌套层数，扰动参数控制各层之间的影响强度。

与基于噪声的纹理不同，魔法纹理完全基于解析三角函数，计算确定性强，能产生独特的、非自然的程序化图案。

## 核心函数

### `svm_magic(p, scale, n, distortion)`
- **功能**: 计算魔法纹理的 RGB 颜色值
- **参数**:
  - `p`: 3D 坐标
  - `scale`: 缩放因子（乘以 5.0 作为基础频率）
  - `n`: 深度(depth)，控制嵌套层数（0-10）
  - `distortion`: 扰动值，控制各层间的影响强度
- **返回**: `float3` RGB 颜色，各通道在约 [0, 1] 范围

### `svm_node_tex_magic(kg, stack, node, offset)`
- **功能**: SVM 节点入口函数
- **输入**: 坐标(co)、缩放(scale)、扰动(distortion)、深度(depth)
- **输出**:
  - **因子(Fac)**: RGB 三通道的平均值 `average(color)`
  - **颜色**: 魔法纹理 RGB 颜色

## 实现细节

### 初始值计算
三个通道的初始值通过三组不同符号组合的三角函数生成：
```cpp
x = sin((px + py + pz) * 5 * scale)
y = cos((-px + py - pz) * 5 * scale)
z = -cos((-px - py + pz) * 5 * scale)
```
坐标预先取模 `2*pi` 以防止大坐标值导致的 NaN 和精度损失。

### 嵌套迭代
从深度 0 到最大 10，每一层通过以下模式更新 x/y/z 值：
1. 将当前值乘以 `distortion`
2. 使用 `cos` 或 `sin`（正负交替）计算新值，参数为 x/y/z 的不同线性组合
3. 新值再乘以 `distortion`

每增加一层深度，图案的复杂度和频率都会增加，产生越来越精细的干涉条纹。

### 输出归一化
若 `distortion != 0`，最终 x/y/z 除以 `2 * distortion`，然后偏移到 `0.5 - value` 使输出中心化在 [0, 1] 附近。

### NaN 防护
代码注释详细说明了输入坐标的 NaN 防护：
- `(px + py + pz) * 5` 可能在某个坐标极大时溢出为 Inf，而 `sin(Inf)` 返回 NaN
- 一个维度远大于其他维度时，浮点精度损失会导致条带伪影
- 通过 `fmodf(p, 2*pi)` 将输入限制在合理范围内解决这两个问题

### 深度的条件展开
深度检查使用嵌套 `if (n > i)` 而非循环，这可能是为了帮助 GPU 编译器进行更好的优化（避免动态循环的寄存器分配问题）。最大深度为 10 层。
