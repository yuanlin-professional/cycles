# checker.h - 棋盘格纹理 SVM 节点

## 概述
`checker.h` 实现了 Cycles 的棋盘格纹理节点(`NODE_TEX_CHECKER`)，生成经典的黑白交替棋盘格图案。这是最简单的程序化纹理之一，基于三维空间中各坐标轴整数部分的奇偶性进行判断，输出二值结果（0 或 1）以及对应的两种颜色。

该实现简洁高效，适合作为遮罩、UV 检测或简单的程序化着色使用。

## 核心函数

### `svm_checker(p)`
- **功能**: 计算 3D 空间中点 `p` 的棋盘格值
- **返回**: `float`，1.0 或 0.0
- **算法**:
  1. 对坐标添加微小偏移 `+0.000001` 并缩放 `*0.999999`，避免整数边界上的精度问题
  2. 对各轴取 `floor` 后转为绝对值整数：`xi = abs(floor(p.x))`
  3. 检查三轴奇偶性：`(xi%2 == yi%2) == (zi%2)` 为真返回 1.0，否则返回 0.0

### `svm_node_tex_checker(stack, node)`
- **功能**: SVM 节点入口函数
- **输入**: 坐标(co)、颜色1(color1)、颜色2(color2)、缩放(scale)
- **输出**:
  - **颜色**: `f == 1.0` 时输出 `color1`，否则输出 `color2`
  - **因子(Fac)**: 棋盘格值（0.0 或 1.0）

## 实现细节

### 精度保护
坐标在 floor 操作前进行微调 `(p + 0.000001) * 0.999999`，目的是避免当坐标恰好为整数时（如 1.0, 2.0），由于浮点舍入导致的不确定结果。这种偏移足够小，不会在视觉上产生可察觉的偏差。

### 三维奇偶性检查
棋盘格模式本质上是三维空间的奇偶校验：
- 若三个轴的整数部分中，偶数个数为偶数（0 或 2 个），则结果为 1
- 否则为 0
逻辑表达式 `(xi%2 == yi%2) == (zi%2)` 等价于 `(xi + yi + zi) % 2 == 0`，但使用当前写法可能有更好的编译器优化。

### 无额外节点读取
与大多数纹理节点不同，棋盘格节点足够简单，所有参数偏移量和默认值都打包在主 `uint4 node` 中，无需调用 `read_node` 读取额外数据，因此函数签名中没有 `offset` 参数返回值。
